# Founder Skills OS — Coding Host Priority

This is the current implementation priority for Founder Skills OS.

## Current focus

We are intentionally prioritizing **coding harnesses and orchestrated coding environments** over chat-native bundles.

### Primary hosts right now
- pi
- Claude Code
- Codex
- OpenCode
- OpenClaw
- Hermes

### Deferred for now
- ChatGPT prompt bundles
- Claude chat prompt bundles

These still matter long-term, but the immediate value is in making the coding-host and orchestrator story exceptionally strong.

---

## Why this is the right move

Founder Skills OS must not feel like:
- great for strategy
- weak for shipping

So the near-term product should be strongest where founders feel the most risk:
- turning scope into implementation
- reviewing architecture
- verifying quality
- auditing release readiness
- keeping build work connected to launch, sales, PMF, and support

---

## OpenClaw strategy

OpenClaw is not just another skill folder target.
It is an orchestrator.

So Founder Skills OS should support OpenClaw through:
- AGENTS section generation
- lite/full coding session prompt templates
- sequence-aware spawn instructions
- founder-partner routing at orchestration time

This is inspired by gstack's best OpenClaw ideas, but the improvement is that the orchestration logic is based on:
- company bottlenecks
- founder state
- artifact graph
- startup operating sequences

not just coding workflow discipline.

---

## Hermes strategy

Hermes appears to support a native skills directory at:
- `~/.hermes/skills/`

So Founder Skills OS should treat Hermes as a first-class coding-agent host with:
- native `SKILL.md` generation
- install docs for Hermes skill paths
- sequence support
- founder-partner routing
- engineering + product + GTM workflows available in one place

This gives Founder Skills OS another strong distribution surface beyond Claude Code-style runtimes.

---

## What “better than gstack” means here

Steal from gstack:
- host generation rigor
- OpenClaw orchestration patterns
- strong coding workflow discipline
- build / review / QA / release thinking

Improve with Founder Skills OS:
- company-level bottleneck routing
- engineering + product tied to founder goals
- GTM, SEO/GEO, ads, support, hiring, and weekly operations in the same system
- host-agnostic canonical source
- richer company memory and artifact graph

That is the strategy.
